# Yoora Sarah Business Glossary

## Canonical Terms for Website, Portal, and AI Delivery

| Field | Value |
| --- | --- |
| Date | 2026-04-22 |
| Status | Phase 0 draft |
| Purpose | Canonical vocabulary for product, commerce, operations, management, SDM, and AI |

## 1. Surface and User Terms

| Term | Definition |
| --- | --- |
| Website | Public-facing buyer and consumer surface in `apps/web` |
| Portal | Internal management and SDM surface in `apps/portal` |
| Buyer AI | AI assistant features used by buyers on the website |
| Internal AI | AI assistant features used by management, SDM, and employees in the portal |
| Management | Leadership and operational decision-makers using portal workflows |
| SDM | HR and employee-enablement functions including SOP, onboarding, and internal policy access |
| Assisted Selling | Human-assisted support via WhatsApp or similar channels to help conversion |

## 2. Commerce Terms

| Term | Definition |
| --- | --- |
| Catalog | The structured set of products, categories, variants, collections, and launch metadata |
| Collection | A grouped commercial or campaign release tied to a theme, season, or launch |
| Edit | A curated route through products based on mood, occasion, styling intent, or commercial focus |
| PDP | Product detail page in the website storefront |
| Trust Layer | The combination of content, UI states, policy visibility, and support cues that reduce buyer hesitation |
| Launch | A planned release of a collection or commercial highlight |
| Preorder | A purchasable product or collection state fulfilled after the purchase date |
| Support Handoff | Escalation from AI or structured UX into human CS or assisted selling |

## 3. Operations Terms

| Term | Definition |
| --- | --- |
| Brief | Structured input for design, marketing, or campaign work |
| Approval | Controlled review step for business, design, or operational decisions |
| Forecast Run | A discrete planning run that produces demand recommendations |
| Forecast Recommendation | Suggested quantity, size mix, color mix, or allocation derived from planning logic |
| Production Plan | The internal plan translating forecast signals into production action |
| Audit Trail | Logged record of sensitive or reviewable actions across approvals, AI, and workflow events |

## 4. AI Terms

| Term | Definition |
| --- | --- |
| AI Styling | Buyer-facing guidance for finding products, looks, moods, or combinations |
| Support Retrieval | Tool-backed factual answers for policy, order, stock, shipping, and support questions |
| Knowledge Assistant | Portal-native AI that retrieves SOPs, policy, onboarding, and internal knowledge |
| Copilot | Context-aware internal AI helper embedded into a workflow such as briefs or approvals |
| Tool | A governed API or service capability used by AI to answer with business-backed data |
| Human-in-the-Loop | Mandatory human review or takeover for sensitive, ambiguous, or high-impact actions |
| AI Log | Stored trace of prompts, tool calls, outputs, and evaluation metadata for AI interactions |

## 5. Delivery Terms

| Term | Definition |
| --- | --- |
| Phase | Program-level delivery grouping used in the master timeline |
| Sprint | Two-week implementation unit used in the technical backlog |
| Ticket | Implementation-ready unit of work linked to a module and likely target files |
| Gate | Required approval or readiness checkpoint before moving to the next phase |
| Baseline UI/UX | The current approved Yoora Sarah website experience that must be preserved unless redesign is explicitly approved |

## 6. Naming Rules

- Use `website` for buyer-facing commerce surface.
- Use `portal` for management and SDM internal surface.
- Use `buyer AI` for website-side AI capabilities.
- Use `management and SDM AI` or `internal AI` for portal-side AI capabilities.
- Do not use `redesign` to describe current execution scope.
- Use `alignment`, `refinement`, `trust improvement`, `workflow clarity`, or `AI enablement` for current delivery work.
